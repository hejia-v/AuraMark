# sprites-core-symbols 图标索引

共 541 个 SVG，文件名为哈希值，viewBox 以 `0 0 20 20` 为主。

## 在 WPF 中使用

这些 SVG 是单色路径图标（无 CSS 变量，无 stroke），可直接提取 `d` 属性用于 WPF：

```csharp
var icon = new System.Windows.Shapes.Path
{
    Data = Geometry.Parse("<从 SVG 的 d 属性复制的路径数据>"),
    Fill = new SolidColorBrush(Color.FromRgb(0x9A, 0xA4, 0xB0)),
    Width = 14, Height = 14,
    Stretch = Stretch.Uniform,
    VerticalAlignment = VerticalAlignment.Center,
};
```

注意：需要 `using Orientation = System.Windows.Controls.Orientation;` 等别名以避免与 WinForms 冲突。

## 按特征快速筛选

### 纯色单路径图标（1 个路径，无 stroke，无 CSS 变量，viewBox=20x20）
约 400+ 个，适合直接用于 WPF。

### 彩色文件类型图标（多路径 + CSS 变量 `var(--xxx)`）
用于 Web 前端显示文件类型，不适合直接用于 WPF（需替换颜色）。
代表：`e7a7d8.svg`（视频）、`1018ef.svg`（PDF）、`067e94.svg`（ZIP）、`2fdcf8.svg`（代码文件）、`ccfd18.svg`（图片）等。

### stroke-only 图标（`stroke="currentColor"`，`fill="none"`）
需设置 WPF `Path.Stroke` 而非 `Fill`：`53a0d8.svg`、`03583c.svg` 等。

### 小尺寸图标（12x12 或 16x16）
`070314.svg`（chevron up, 12x12）、`111434.svg`（chevron down, 12x12）、
`0f07fb.svg`（play, 12x12）、`53e928.svg`（expand, 16x16）、`e51fba.svg`（arrow, 16x16）

## 如何识别未知图标

1. 读取 SVG 内容，检查路径数量：
   - 1 路径 + 无 CSS 变量 -> 纯色单图标，直接用
   - 多路径 + `var(--xxx)` -> 彩色文件类型图标
2. 将 `d` 属性内容粘贴到 https://yqnn.github.io/svg-path-editor/ 预览形状
3. 提取后在 WPF 中用 `Geometry.Parse()` + `System.Windows.Shapes.Path` 渲染
